Multithreading

What is a thread?

A thread is a path for an execution of a programm. The programm is organized to run a multiple number of threads parallel. The big advance is, that a programm is not interrupted, when it gets for example information from another source. It is not to be mistaken with multitasking, where several applications run side by side. The big difference is, that multithreading allows you to run multiple parts of one programm run side by side.

Fields of application

  • Internet applications
    • Image Buildup
    • Queries
  • Server applications
    • Several clients parallel

Usage of multithreading with Java

Java supports multithreading. It offers predefined classes, which can be used by the programmer to create and change threads. There are two ways to create threads in Java.

  • It is possible to implement a class which derives from a predefined class "Thread".
  • If you want to create a thread of a class, which already derives from a class you have to use another way, because Java doesn't support multiple inheritance. Therefor exists the possibility to implement the "runnable"-Interface.
These are the most important ways to controll a thread:
  • Start a thread
    • After a thread was created, it has to be starte with the method "start()" of the class "Thread". For example with: nameofthread.start().
  • Pause a thread
    • To pause a thread you unfortunately can't use a command like "nameofthread.stop()". Therefor you have to look at your run()-method. If this method comes to its end, the thread will stop.
