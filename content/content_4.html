Basic Datatypes

Variables are reserved memory locations to store values. When you create a variable you reserve some space in memory.
Based on the data type of a variable, the operating system allocates memory and decides what can be stored in the reserved memory. Therefore, by assigning different data types to variables, you can store integers, decimals, or characters in these variables. There are 2 different data types in Java:

  • Object Data Types
  • Primitiv Data Types

Primitiv Data Types

There are 8 different kinds of Primitive Data Types. They are predefined by the language and named by the keyword.
byte
- 8-bit signet two's complement integer
- minimum value: -2^7
- maximum value: 2^7-1
- default value: 0
- Byte data type is used to save space in large arrays, mainly in place of integers, since a byte is four times smaller than an int.
short
- 16-bit signet two's complement integer
- minimum value: -2^15
- maximum value: 2^15-1
- default value: 0
- Short data type can also be used to save memory as byte data type. A short is 2 times smaller than an int
int
- 32-bit signed two's complement integer
- minimum value: -2^31
- maximum value: 2^31-1
- default value: 0
Int is generally used as the default data type for integral values unless there is a concern about memory.
long
- 64-bit signed two's complement integer
- minimum value: -2^63
- maximum value: 2^63-1
- default value: 0L
- Long is used, when an int is to short and you need a wider range
float
- single-precision 32-bit IEEE 754 floating point
- used to save memory in large arrays of floating point numbers
- float data types is never used for precise values
- default value: 0.0f
double
- double-precision 64-bit IEEE 754 floating point
- default value: 0.0d
- usually used as default value for decimal values
- one bit of information
- values: true or false
- default value: false
char
- single 16-bit unicode character
- minimum value: 0
- maximum value: 65,535
- used to store any charecter
