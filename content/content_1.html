Platform Independence

What is the meaning of platform independence?

Platform independent, cross-platform or multi-platform in computing means that you can run code with little or no modification on multiple platforms.

Platform Independent software can be divided into two types:

  • The first requires individual building or compilation for each platform that it supports
  • The other one can be directly run on any platform without special preparation

How is Java platform independent?

Java has a virtual machine the so-called JVM short for Java Virtual Machine.

What happens in case of Java is that the JVM once installed on any platform like windows or OS X can run the java code without any alteration.
The JVM acts like a virtual platform on which the code is executed.
As the platform being JVM remains constant throughout all platforms Java programs can run on any platform irrespective of the real platform.

Java source code is first converted to javabyte code.
This byte code is not readable by the original platform, but only the JVM can read it and then it executes it.

JVMs are available for most well known Operating Systems like Windows, OS-X, Linux etc - so once you've developed your software in Java you can run it on any of these platforms.
This is the primary reason why Java is much in use even today.

Java Virtual Machine

The JVM itself needs specific versions for each platform to take account of different native instruction codes and machine capabilities (so you can't run a Windows JVM on Linux and vice Versa).
