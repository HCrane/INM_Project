Collections

Whenever you need more than one object as a whole, you need a collection. You frequently handle a lot of objects together to:
  • group, assemble, relate, order them in hierarchies
  • process them all at once (e.g. count, update, etc.)
  • store them in a database or files (read, write)
  • transmit them via network
A collection (container in other programming languages e.g. C#) represents a group of objects, called members or elements with two relevant criteria:
uniqueness
- is each member unique within its collection, or are dublicates allowed
ordering
- are all members ordered within its collection at every time
With respect to these two criteria, it builds an architecture to represent and and process collection in an uniform way. It is based on:
interfaces
- to declare collections (ADTs)
implementations
- for all interfaces
algorithms
- in form of methods to process them ( insert, delete, search, sort,...)

Example: Set of Strings


  import java.util.*;
  public class SetOfStrings{

  	public static void main(String[] args) {
  		HashSet s = new HashSet();
  		s.add("Manfred");
  		s.add("Manfred");
  		s.add("Patrick");
  		s.add("Anna");
  		System.out.println(s);
  		System.out.println(new TreeSet(s));
  	}
  }

Summary:

A collection is an object that can hold references to other objects.

The collection interfaces declare the operations that can be performed on each type of collection.
